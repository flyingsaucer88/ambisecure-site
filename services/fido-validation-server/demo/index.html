Demo · access posture

FIDO Validation Server — demo provisioning in progress.

The live demo runtime at fido.ambisecure.ambimat.com is being provisioned for early-access reviewers. To keep the surface narrow and audit-friendly, access is request-based rather than self-serve during this phase.

01

What the demo will show

A working FIDO2 registration + authentication ceremony, attestation verification against an MDS mirror, and per-tenant policy enforcement — using the exact server image documented in the service page.

02

What we ask of reviewers

Tell us your tenant scope, target authenticators (security key / passkey / SIM applet), and integration language. We'll provision a sandbox tenant + API keys within one business day and walk through the ceremony with you.

03

No data leaves the demo

Demo tenants are isolated, time-bounded, and torn down after the review window closes. No credentials, no test users, no relay metadata persists past the demo.

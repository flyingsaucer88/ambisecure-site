Tool · NDEF

NDEF Personalisation Tool

Programs the NDEF payload — URL, vCard, or custom record — onto the contactless surface of an AmbiSecure card or token. Designed for issuer use during personalisation, not for end-user reprogramming.

NDEFNFC ForumvCardURL
Request engagement Tappable product
What the tool does

Issuer-side NDEF programming for AmbiSecure form factors.

URL records

Program a URL record — e.g. https://card.example.com/u/abc123 — that resolves to the user’s profile or a registration link on tap.

vCard records

Program a vCard record so a tap exchanges contact information directly into the phone’s contacts app.

Custom records

Custom NFC Forum NDEF record types for application-specific payloads (loyalty, ticket, voucher).

Bulk programming

Drives a stack of cards in parallel. Per-card unique payload via CSV or REST.

Audit

Per-card programming log with chip serial, payload hash, and outcome.

Lock policy

Optional NDEF lock-bit programming to prevent post-issuance modification.
